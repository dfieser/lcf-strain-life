metadata:
  type: feedback
---

All prose written for this project must avoid em dashes, avoid semicolons, and avoid unnecessary parentheses. This covers READMEs, docs, code comments, docstrings, commit messages, and chat replies. Hyphens are fine.

**Why:** The user wants clean, direct, human sounding writing, without the punctuation patterns that read as machine generated.

**How to apply:** Replace em dashes with a period, a comma, or a colon. Replace prose semicolons with two sentences or a comma. Remove parenthetical asides or turn them into commas, keeping only genuinely useful labels such as units or file paths. The same rule is hardcoded in CLAUDE.md. See [[project-scope-lcf-automation]].
